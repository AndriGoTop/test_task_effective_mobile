# Тестовое задание Effective Mobile

## API

### Пользователи

    POST /sign_up - регистрация пользователя
    POST /token - получение jwt токена для аутентификации сессии
    POST /logout - logout пользователя путем блокировки jwt токена
    DELETE /users - удаление пользователя, оставляя его в БД со статусом is_active=FAlse
    PATCH /users - обновление данных пользователя
    GET /users/me - получение данных активного пользователя

### Роли

    PUT /role - обновление данных роли
    POST /role - создание роли

### Посты

    POST /posts - создание поста
    GET /posts - чтение новостей
    PUT /posts - изменение новости
    DELETE /posts - Удаление новости 

## Роли

Разграничение ролей и прав пользователя происходит следующим образом: пользователь создаёт роль, где указывает
возможности обладателя (создание, удаление, изменение, чтение), при регистрации пользователя указывается его роль, затем
при отправке запроса на работу с постами, к примеру на чтение, идёт проверка того имеет ли пользователь с данной ролью
право на данный запрос если не имеет, то ответом будет ошибка 403, если имеет то запрос будет выполнен

## Запуск программы
В корневой папке проекта должен находиться .env файл следующего вида:
```dotenv
POSTGRES_PASSWORD=<Пароль_пользователя>
POSTGRES_USER=<Имя_пользователя>
POSTGRES_DB=<Имя_БД>
POSTGRES_HOST=db
POSTGRES_PORT=5432
ALGORITHM=HS256
```
Быстрый копипаст
```dotenv
POSTGRES_PASSWORD=Password123
POSTGRES_USER=postgres
POSTGRES_DB=auth_em
POSTGRES_HOST=db
POSTGRES_PORT=5432
ALGORITHM=HS256
```
Запуск docker-compose
```commandline
docker compose up -d
```
